What is this System?

This panel demonstrates the Advanced Digital Transformation (ADT) approach in action, blending governance-native protocols with the latest development technologies and innovative solutions.

While specialized AI Virtual Agents serve as the project's workforce, the core focus is the framework itself—ensuring compliance, transparency, and architectural integrity by design.

Virtual Workforce

The project is built by 10 specialized AI roles working in concert. No single agent "does everything." Instead, they delegate based on expertise:

Leadership
Systems Architect, Overseer, Product Manager
Engineering
Embedded, Electrical, Mechanical, Network
Software
Backend, Frontend, DevOps
ADT Principles (Paul Sheridan)

The system adheres to Governance-Native development. Governance is not a meeting; it is code.

  • The ADS (Authoritative Data Source): A single, append-only ledger (events.jsonl) that records every action. If it's not in the ADS, it didn't happen.
  • Compliance by Construction: Agents cannot "break the rules" because the rules are part of their runtime environment. Unauthorized actions are physically blocked.
  • Transparency: This dashboard renders the ADS in real-time. You see exactly what the AI sees. No hidden states.
The Practices
_cortex
The "brain" folder of the project. Contains the memories, plans, specs, and the ADS ledger. It is the shared consciousness of the Hive Mind.
Spec-Driven Development (SDD)
A strict rule: "No Spec, No Code." Agents must write a Specification Document (Spec) and get approval before writing a single line of implementation code.
The Overseer
A specialized role dedicated solely to compliance, logging, and truth-telling. The Overseer does not build; it watches.
Hive Mind
The architecture where multiple distinct AI agents (Claude, Gemini) access the same file system and context to work as a single team.
